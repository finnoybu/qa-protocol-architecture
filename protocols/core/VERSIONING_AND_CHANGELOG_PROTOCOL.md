# VERSIONING AND CHANGELOG PROTOCOL
Version: 1.0.0

## Purpose

Define how QA protocols evolve without loss of determinism.

---

## Version Format

MAJOR.MINOR.PATCH

- MAJOR = Structural change
- MINOR = New validation layer added
- PATCH = Wording correction or clarification

---

## Change Requirements

All changes must include:

- Version increment
- Short rationale
- Date
- Author

---

## Backward Compatibility Rule

Protocols must not remove validation requirements without explicit documentation.

---

## Git Policy

- All changes committed with descriptive message.
- No force pushes to main.
- Tag major versions.

---

End of Document
